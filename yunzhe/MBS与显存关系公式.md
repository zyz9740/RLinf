# MBS 与 GPU 显存关系公式

基于 37 个实验的实测数据分析

---

## 📐 核心公式

### **训练阶段 GPU 2-3 (Actor/Training) 显存**

```
Memory(GB) = 18.30 + 0.0503 × MBS
```

或

```
Memory(MB) = 18736 + 51.54 × MBS
```

**适用范围**: MBS ∈ [1, 32]  
**拟合度**: R² = 0.503

---

## 📊 分环境数的精确公式

不同环境数下，MBS 的影响程度不同：

| env | 公式 (MB) | R² | MBS 影响 |
|-----|-----------|-----|----------|
| 2   | 19302 + 7.08 × MBS | 0.04 | 很弱 (+1.1%) |
| 4   | 18897 + 31.78 × MBS | 0.73 | 中等 (+5.2%) |
| 8   | 18186 + 48.62 × MBS | 0.91 | **强** (+8.3%) |
| 16  | 18397 + 29.56 × MBS | 0.41 | 中等 (+5.0%) |
| 32  | 18034 + **90.53** × MBS | 0.89 | **很强** (+15.6%) |
| 64  | 17875 + **101.66** × MBS | 0.86 | **极强** (+17.6%) |

**观察**: env 越大，MBS 对显存的影响越显著

---

## 🎯 实际应用场景

### 场景 1: 小环境数 (env ≤ 8)

```
Memory(GB) ≈ 18.2 + 0.03~0.05 × MBS
```

- MBS 影响较小，可以自由调整 MBS 优化训练速度
- 推荐配置: **MBS=16~32**

### 场景 2: 大环境数 (env ≥ 32)

```
Memory(GB) ≈ 17.9 + 0.09~0.10 × MBS
```

- **MBS 影响显著**，需要谨慎选择
- MBS=32 时会额外占用 **3GB** 显存
- 推荐配置: **MBS=4~8** (平衡性能和显存)

## 🔍 深入分析

### 1. 分段拟合

MBS 在不同范围内的影响不同：

- **MBS ∈ [1, 16]**: `Memory = 18656 + 10.30 × MBS` (R²=0.017, 影响很弱)
- **MBS ∈ [16, 32]**: `Memory = 17367 + 92.77 × MBS` (R²=0.583, 影响显著)

**结论**: MBS 在 16 附近存在**拐点**，超过 16 后显存增长加速

### 2. 为什么大 env 下 MBS 影响更大？

**理论解释**:
```
训练显存 = 模型参数 + 优化器状态 + 梯度 + 激活值缓存

激活值缓存 ∝ MBS × model_layers × hidden_dim
```

- 小 env: GPU 显存充足，激活值可以全部保留
- 大 env: Rollout 已占用大量显存，激活值缓存受到更严格限制
- 大 MBS: 需要保留更多中间激活值用于反向传播

### 3. 实测数据验证

以 **env=64, MBS=32** 为例：

- 预估显存: 17875 + 101.66 × 32 = **21,128 MB**
- 实测显存: **21,577 MB**
- 误差: +449 MB (2.1%)

✅ 公式准确性验证通过

## ⚙️ 完整显存预估公式

结合 Rollout 和 Training 两个阶段：

### Rollout 阶段
```
GPU 0-1: Memory(GB) = 7.63 + 0.370 × env
GPU 2-3: Memory(GB) ≈ 17.5 (固定)
```

### Training 阶段
```
GPU 0-1: Memory(GB) ≈ 与 Rollout 相近
GPU 2-3: Memory(GB) = 18.30 + 0.0503 × MBS
```

### 综合预估 (取峰值)
```
GPU 0-1: 7.63 + 0.370 × env  (GB)
GPU 2-3: max(17.5, 18.30 + 0.0503 × MBS)  (GB)
```

## 💡 配置建议

### 根据 GPU 显存选择 MBS

假设 GPU 显存限制为 **46 GB** (L40):

1. **env=2**:  
   - GPU01: 8.4 GB, GPU23: 18.3~20 GB  
   - 可选 MBS: **任意** (1~256)

2. **env=16**:  
   - GPU01: 13.6 GB, GPU23: 18.3~22 GB  
   - 可选 MBS: **≤64**

3. **env=32**:  
   - GPU01: 19.4 GB, GPU23: 18.3~24 GB  
   - 可选 MBS: **≤32**

4. **env=64**:  
   - GPU01: 31.3 GB, GPU23: 18.3~22 GB  
   - 可选 MBS: **≤32** (显存紧张)

5. **env=96**:  
   - GPU01: 43.2 GB (接近上限)  
   - 可选 MBS: **≤16** (极度紧张)
   - ⚠️ 实测 env=96 会 OOM，不推荐使用

## ⚠️ 使用注意事项

1. **仅适用于训练阶段 GPU 2-3**  
   Rollout GPU (GPU 0-1) 主要受 env 影响，与 MBS 无关

2. **线性假设的局限性**  
   MBS > 32 的公式为线性外推，实际可能有非线性效应

3. **建议预留 buffer**  
   实际配置时建议预留 **10-20% 显存 buffer** 处理峰值

4. **不同模型可能不同**  
   此公式基于 OpenPI 模型，其他模型的系数可能不同

5. **GBS 固定为 256**  
   所有实验都使用 GBS=256，如果修改 GBS，公式可能不适用
