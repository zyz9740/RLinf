# Yunzhe 目录工作报告

## 一、工作概述

完成了训练性能优化实验的监控与分析工具链，包括系统资源监控和性能分析。

## 二、主要工作内容

### 1. 系统监控工具 (monitor/)

实时采集训练过程的 GPU、CPU、内存资源占用，生成可视化报告。

- 按秒级采样资源使用情况
- 输出 CSV 数据和 HTML 可视化报告
- 支持后台运行和中断保存

### 2. 实验分析工具 (analyze_experiments.py)

自动整合训练日志与监控数据，生成对比分析报告。

**提取的关键指标：**
- 性能：Rollout 时间、Step 时间、训练时间
- 资源：GPU 显存占用、GPU 利用率、CPU/内存占用
- 阶段分析：区分 Rollout 和 Training 阶段的资源使用

**分析功能：**
- 交互式表格，支持排序
- 横向对比不同配置的性能
- 识别 OOM 边界和性能瓶颈

### 3. 实验数据

- **48+ 组实验**：覆盖不同环境数量（1-64）和批次大小（1-32）
- **两种模式**：split 和 all 两种 GPU placement 模式
- **完整数据**：训练日志 + 资源监控时序数据

### 4. 分析结果

- **HTML 报告**：4 个视图对比性能和资源，可交互排序
- **CSV 数据**：所有指标的结构化数据，支持二次分析

## 三、应用场景

1. **配置优化**：找出训练速度最快的参数组合
2. **资源规划**：确定不同配置下的显存需求
3. **模式对比**：评估 split vs all 的效率差异
4. **瓶颈定位**：识别性能瓶颈环节


## 五、总结

建立了完整的训练性能评估工具链，通过 48+ 组参数扫描实验，为训练配置优化提供了数据支撑。
