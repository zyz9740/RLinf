## 二、增长规律分析

### 1. 环境数翻倍时的显存增长

| 环境数变化 | 稳定显存增长 | 规律解释 |
|-----------|-------------|---------|
| 2 → 4     | +10.4%      | 小幅增长 |
| 4 → 8     | +16.4%      | 加速增长 |
| 8 → 16    | +26.9%      | 持续加速 |
| 16 → 32   | +43.5%      | 显著加速 |
| 32 → 64   | +61.3%      | 接近线性 |

**平均每翻倍增长：31.7%**

**增长模式判断：**
- 如果是**线性增长**（mem ∝ env）：翻倍时增长 100%
- 如果是**平方根增长**（mem ∝ √env）：翻倍时增长 41.4%
- **实测 31.7%**：介于对数增长和平方根增长之间
- **趋势**：随着 env 增大，增长率从 10% 加速到 61%，逐渐接近线性

## 三、拟合模型

### 线性模型（峰值显存）

```
显存(MB) = 265.0 × env数 + 14113
```

- **R² = 1.0000**（近乎完美拟合）
- **物理意义：**
  - 基础开销：14.1 GB（模型权重、系统缓冲等）
  - 增量开销：265 MB/env

### 为什么"翻倍增长"不是 100%？

因为基础开销（14.1 GB）的存在：

```
env=2:  显存 = 265×2  + 14113 = 14643 MB
env=4:  显存 = 265×4  + 14113 = 15173 MB  → 增长 3.6%
env=64: 显存 = 265×64 + 14113 = 31073 MB
env=128:显存 = 265×128+ 14113 = 48033 MB  → 增长 54.6%
```

当 env 很小时，基础开销占主导，增长率低。  
当 env 很大时，增量开销占主导，增长率接近 100%（线性）。

## 六、核心结论

### ✓ 线性增长确认

GPU01 显存与环境数量呈**近乎完美的线性关系**：

```
显存(GB) ≈ 0.26 × env数 + 14
```

- R² = 1.0000（线性拟合）
- 边际成本稳定在 380 MB/env
- 早期因基础开销影响显示"次线性"，实质是线性

### ✓ 容量限制

| 配置 | 环境数上限 | 显存占用 | 适用场景 |
|------|-----------|---------|---------|
| 🟢 安全 | ≤ 64 | 31 GB | **推荐生产环境** |
| 🟡 激进 | 65-90 | 32-38 GB | 短期实验 |
| 🟠 风险 | 91-100 | 38-41 GB | 极限测试 |
| 🔴 危险 | ≥ 101 | >41 GB | 必定失败 |

### ✓ 优化方向

**降低基础开销（14 GB）：**
1. 模型量化（FP16 → INT8）
2. 梯度检查点（Gradient Checkpointing）
3. 卸载不活跃的模型层到 CPU

**降低边际成本（380 MB/env）：**
1. 压缩环境状态表示
2. 共享观测缓冲区
3. On-the-fly 数据增强（减少存储）

**突破硬件限制：**
1. 分布式环境（多机多卡）
2. CPU-GPU 混合存储
3. 更大显存的 GPU（A100: 80GB, H100: 80GB）
